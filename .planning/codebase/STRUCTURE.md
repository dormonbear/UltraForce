## Directory Purposes

**`src/background/`:**
- Purpose: Plasmo service worker entry point
- Contains: Message handlers for session/cookie retrieval, extension install setup
- Key files: `src/background/index.ts`

**`src/contents/`:**
- Purpose: Scripts injected into matched Salesforce pages; each file becomes a separate content script
- Contains: UI orchestration, keyboard suppression, Setup page enhancement
- Key files: `src/contents/salesforce-search.tsx`, `src/contents/keyboard-shield.ts`, `src/contents/setup-enhancer.ts`

**`src/components/`:**
- Purpose: Pure React presentation components; no direct Chrome API calls
- Contains: Search modal component tree; one error boundary at root
- Key files: `src/components/search/SearchModal.tsx`, `src/components/search/SettingsPanel.tsx`, `src/components/ErrorBoundary.tsx`

**`src/lib/`:**
- Purpose: All business logic — API, caching, search indexing, navigation, settings
- Contains: Utilities, service classes, the WindowManager orchestrator
- Key files: `src/lib/window-manager.ts`, `src/lib/salesforce-api.ts`, `src/lib/auth.ts`, `src/lib/fuzzy-search.ts`, `src/lib/metadata-cache.ts`

**`src/types/`:**
- Purpose: Centralized TypeScript type definitions shared across all layers
- Key file: `src/types/index.ts` — exports `SearchResult`, `NavigationMode`, `RecordContext`, `BuiltinCommand`, `CustomCommand`, `SearchCommand`, type guards

**`tests/e2e/`:**
- Purpose: Playwright E2E test suites requiring a live Salesforce org (`ultraforce` sf CLI alias)
- Contains: Browser-based tests for search and navigation flows

## Key File Locations

**Entry Points:**
- `src/background/index.ts`: Background service worker
- `src/contents/salesforce-search.tsx`: Main content script (UI + orchestration)
- `src/contents/keyboard-shield.ts`: MAIN-world keyboard patch (runs before page scripts)
- `src/contents/setup-enhancer.ts`: Setup page lazy-loader

**Configuration:**
- `tsconfig.json`: Path aliases — `~lib/*`, `~types`, `~components/*` map to `src/` subdirs
- `package.json`: Plasmo manifest config (permissions, host patterns, background/content scripts)
- `vitest.config.ts`: Test environment (jsdom), setup file (`src/test-setup.ts`)
- `playwright.config.ts`: E2E browser config (headed required for extensions)

**Core Logic:**
- `src/lib/window-manager.ts`: Singleton state manager and React lifecycle controller
- `src/lib/salesforce-api.ts`: All Salesforce metadata queries and search orchestration
- `src/lib/auth.ts`: Session cookie retrieval and authenticated HTTP client
- `src/lib/fuzzy-search.ts`: MiniSearch index management
- `src/lib/metadata-cache.ts`: Persistent cache layer
- `src/lib/command-parser.ts`: Search command syntax definitions and parser

**Type Definitions:**
- `src/types/index.ts`: All shared types; import via `~types`

**Testing:**
- `src/lib/*.test.ts`: Unit tests co-located with source files
- `src/test-setup.ts`: Global Vitest setup (chrome API mocks)
- `tests/e2e/`: Playwright E2E tests in separate directory

## Naming Conventions

**Files:**
- `kebab-case.ts` for lib modules: `fuzzy-search.ts`, `command-parser.ts`, `window-manager.ts`
- `PascalCase.tsx` for React components: `SearchModal.tsx`, `ResultItem.tsx`, `ErrorBoundary.tsx`
- `kebab-case.test.ts` for unit tests co-located with source: `command-parser.test.ts`, `keyboard-interceptor.test.ts`

**Directories:**
- Lowercase: `background/`, `contents/`, `lib/`, `types/`, `styles/`
- Feature-grouped under `components/search/` (not split by type)

**Exports:**
- Types: `interface` keyword preferred over `type` for object shapes
- Functions: camelCase exported functions; classes use PascalCase
- Type guards: named `is<Type>()` pattern (e.g., `isCustomCommand`, `isBuiltinCommand`)

## Where to Add New Code

**New metadata type search support:**
- Add SOQL query to `METADATA_TYPES` record in `src/lib/salesforce-api.ts`
- Add result formatting in `searchSalesforceMetadata()` in same file
- Add builtin command mapping in `src/lib/command-parser.ts` `BUILTIN_COMMANDS`

**New UI component:**
- Implementation: `src/components/search/<ComponentName>.tsx`
- Import and use in `src/components/search/SearchModal.tsx`
- Add any new styles to `src/components/search/styles.ts`

**New lib utility:**
- Small focused file: `src/lib/<feature-name>.ts`
- Export functions; import in `window-manager.ts` or `salesforce-api.ts` as needed

**New content script:**
- File: `src/contents/<name>.ts` or `src/contents/<name>.tsx`
- Plasmo auto-detects files in `contents/` as content scripts
- Export `config: PlasmoCSConfig` with `matches` and optional `world`/`run_at`

**New type definitions:**
- Add to `src/types/index.ts`; all consumers import via `~types`

**New unit tests:**
- Co-locate with source: `src/lib/<module>.test.ts`
- Use Vitest + `@testing-library/react` for component tests

**New E2E tests:**
- Add to `tests/e2e/`
- See `tests/E2E_TEST_GUIDE.md` for org credentials and test data patterns

## Special Directories

**`.plasmo/`:**
- Purpose: Plasmo build cache and generated manifest artifacts
- Generated: Yes
- Committed: No (in `.gitignore`)

**`.planning/`:**
- Purpose: GSD planning docs (ARCHITECTURE.md, STRUCTURE.md, etc.)
- Generated: By GSD map-codebase command
- Committed: Yes

**`docs/`:**
- Purpose: Human-readable guides — UI design reference, release notes, E2E guide
- Key file: `docs/ui-design-guide.md` — canonical Raycast-style design spec; all UI changes must follow it
