# Phase 03 Plan 01: Centralized Storage Service

**Goal:** Single typed entry point for `chrome.storage.local` replacing all direct usage across `src/`

## Task 1: Create storage service module + tests

Create `src/lib/storage-service.ts` (~150 lines):
- Typed key constants for all 6 fixed keys + `metadataCacheKey(orgId, type)` helper
- Value interfaces per key (settings, error logs, API stats, unsupported types, version check)
- `get<T>(key)`, `set(key, value)`, `remove(key)`, `getAll()` wrapping chrome.storage.local
- `onLocalChanged(callback)` / `offLocalChanged(callback)` wrapping chrome.storage.onChanged

Create `src/lib/storage-service.test.ts` (~200 lines):
- Tests for get/set/remove/getAll delegation
- Tests for onChanged listener filtering to local area
- Tests for metadataCacheKey helper
- Tests for safe defaults on failure

## Task 2: Migrate call sites

Replace direct `chrome.storage.local` in all 10 files:
- metadata-cache.ts, unsupported-types.ts, api-stats.ts, version-check.ts
- window-manager.ts, SearchModal.tsx, salesforce-search.tsx, setup-enhancer.ts
- ErrorBoundary.tsx, background/index.ts

**Verify:** `rg 'chrome\.storage\.local' src/` returns only storage-service.ts

## Success Criteria
1. All chrome.storage.local calls go through storage-service (STAT-02)
2. Fixed keys referenced via typed constants (no magic strings)
3. Dynamic metadata keys via shared helper
4. All 392+ tests pass, zero TS errors, zero lint issues
