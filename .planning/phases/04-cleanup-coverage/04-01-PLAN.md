# Plan 04-01: CSS Migration & Font Bundling

## Goal
Replace CSS-in-JS with a .css file loaded via Plasmo's data-text: import, and bundle Inter font files locally so the extension makes no external network requests for fonts.

## Tasks

### Task 1: Extract CSS to .css file
1. Create `src/components/search/styles.css` from the CSS content in `styles.ts`
2. Remove the `@import url(...)` for Google Fonts from the CSS (handled separately)
3. Update `styles.ts` to re-export from `data-text:./styles.css`
4. Add TypeScript module declaration for `data-text:*` imports

### Task 2: Bundle Inter fonts locally
1. Download Inter woff2 files (weights 300, 400, 500, 600) to `assets/fonts/`
2. Create font loader utility that generates `@font-face` CSS using `chrome.runtime.getURL()`
3. Update SearchModal.tsx to inject font CSS alongside main styles
4. Configure Plasmo `web_accessible_resources` for font files in package.json

## Success Criteria
- [x] `styles.ts` is a thin wrapper importing from `styles.css` via `data-text:`
- [x] No external font requests (no `fonts.googleapis.com` references)
- [x] Inter font loads correctly from local woff2 files
- [x] Build succeeds (`plasmo build`)
- [x] All existing tests pass (437 pass)
- [x] Type-check and lint pass
