Replace the entire hand-rolled `chromeMock` object with vitest-chrome. The new test-setup.ts should:

```typescript
/// <reference types="vitest/globals" />
import '@testing-library/jest-dom/vitest'
import chrome from 'vitest-chrome'

// vitest-chrome provides typed mocks for all Chrome APIs
// but functions have NO default implementation -- configure defaults
// that existing tests depend on
Object.assign(global, { chrome })

// Restore default behaviors (per D-09: maintain existing test expectations)
chrome.storage.local.get.mockResolvedValue({})

chrome.tabs.query.mockResolvedValue([])
```

If the import syntax `import chrome from 'vitest-chrome'` doesn't work, try `import * as chrome from 'vitest-chrome'` and adjust accordingly. The critical requirement is that the global `chrome` object has all the same mock behaviors as the hand-rolled version.
