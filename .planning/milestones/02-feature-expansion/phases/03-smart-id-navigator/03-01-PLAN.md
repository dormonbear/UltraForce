Create `src/lib/id-utils.ts` with:

1. `isSalesforceId(str: string): boolean` — moved from SearchModal.tsx
   Validates 15 or 18 alphanumeric characters.

2. `extractSalesforceId(input: string): string | null` — NEW
   Extracts a Salesforce ID from:
   - Plain ID: `001xxxxxxxxxxxx` → returns the ID
   - Full URL: `https://org.salesforce.com/001xxxxxxxxxxxx` → extracts ID from path
   - URL with Lightning path: `https://org.lightning.force.com/lightning/r/Account/001xxxxxxxxxxxx/view` → extracts ID
   - Mixed text containing an ID: `check record 001xxxxxxxxxxxx please` → extracts ID
   - Returns null if no valid ID found

   Implementation:
   ```typescript
   export function extractSalesforceId(input: string): string | null {
     const trimmed = input.trim()
     if (isSalesforceId(trimmed)) return trimmed

     // Try URL extraction
     try {
       const url = new URL(trimmed)
       const pathSegments = url.pathname.split('/').filter(Boolean)
       for (const segment of pathSegments) {
         if (isSalesforceId(segment)) return segment
       }
     } catch {
       // Not a URL, try regex extraction from mixed text
     }

     // Extract first ID-like pattern from text
     const match = trimmed.match(/\b[a-zA-Z0-9]{18}\b|\b[a-zA-Z0-9]{15}\b/)
     if (match && isSalesforceId(match[0])) return match[0]

     return null
   }
   ```

3. `getKeyPrefix(id: string): string` — returns first 3 characters (key prefix)

4. Export all functions. Remove `isSalesforceId` from SearchModal.tsx and import from `~lib/id-utils` instead.
</action>

### Task 3: Create src/lib/record-preview.ts — Key prefix cache and record fetch

<read_first>
- src/lib/auth.ts (sfRest, API_VERSION, getSession)
- src/lib/salesforce-api.ts (lines 85-95: existing sobjects endpoint usage)
- src/lib/metadata-cache.ts (cache patterns for reference)

</read_first>

<action>
Create `src/lib/record-preview.ts` with:

1. **Key prefix cache** — module-level `Map<string, string>` mapping 3-char prefix to object API name.

   `resolveObjectType(sfHost: string, keyPrefix: string): Promise<string | null>`
   - Check cache first
   - If miss, call `GET /services/data/v62.0/sobjects/` to get global describe
   - Parse response to build full prefix→objectType map, cache all entries
   - Return object type for the requested prefix, or null if not found

2. **Record preview fetch** with LRU cache:

   ```typescript
   interface RecordPreview {
     id: string
     objectType: string
     name: string
     fetchedAt: number
   }
   ```

   Module-level `Map<string, RecordPreview>` with max 50 entries, 5-minute TTL.

   `fetchRecordPreview(sfHost: string, recordId: string, signal?: AbortSignal): Promise<RecordPreview | null>`
   - Check cache (return if fresh)
   - Get key prefix from ID → `resolveObjectType()` to get object type
   - If object type resolved, call `GET /services/data/v62.0/sobjects/{objectType}/{recordId}?fields=Name`
   - Parse response, extract Name (fall back to `Id` if Name not available)
   - Cache and return the preview
   - On 404: return `{ id, objectType: 'Unknown', name: 'Record not found', fetchedAt }` (cached briefly)
   - On 403: return `{ id, objectType: 'Unknown', name: 'No access', fetchedAt }`
   - On network error or AbortError: return null (don't cache)

3. `clearPreviewCache(): void` — for testing

All API calls use `sfRest<T>()` from auth.ts. All errors caught with logger.error.
</action>
